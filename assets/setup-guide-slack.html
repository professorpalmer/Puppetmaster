Puppetmaster — Easy Setup Guide

Routes every task to the cheapest model that can handle it, runs workers as independent processes, and stores results so follow-ups cost $0.

📌 Pin me
Step 1 — Everyone

Install once, wire everything  (idempotent — safe to re-run)

$ pipx install puppetmaster-ai # or: pip install puppetmaster-ai $ puppetmaster setup # doctor + models + MCP installers + rules + hooks
Step 2 — Your platform
⌘

Cursor

MCP is registered by setup automatically — just restart.
$ puppetmaster install-cursor-mcp --global # optional: register for all workspaces
Then: restart Cursor → the agent sees 32+ puppetmaster_* tools. Workspace default writes .cursor/mcp.json.
✳

Claude Code

Needs the claude CLI installed & logged in first.
$ npm i -g @anthropic-ai/claude-code $ claude # complete /login once $ puppetmaster install-claude-mcp
Then: open a fresh Claude session — claude mcp get puppetmaster should show ✔ Connected.
▸

Codex CLI

Needs the codex CLI installed & authenticated first.
$ npm i -g @openai/codex && codex login $ puppetmaster install-codex-mcp
Then: start a fresh Codex session — [mcp_servers.puppetmaster] lands in ~/.codex/config.toml.
⬡

OpenAI API

No MCP needed — used directly as a worker adapter.
$ export OPENAI_API_KEY=sk-... $ puppetmaster route "audit the api" \ --role audit # dry-run the routing
Bonus: billing-grade token telemetry per task. OPENAI_BASE_URL works for compatible providers.

✅ Step 3 — Verify

$ puppetmaster doctor $ puppetmaster show $(puppetmaster last)

🚀 First prompt to try (in Cursor / Claude / Codex)

"Use Puppetmaster to run doctor in this repo and summarize what is missing."
"Use Puppetmaster to start a cursor swarm for this repo and return the job id immediately."
$0 follow-ups: results persist in SQLite — re-read any job with puppetmaster artifacts <job_id>
